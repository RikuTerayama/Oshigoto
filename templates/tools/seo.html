ページを確認する

公開前やSNS共有前に、title、meta description、OGP、canonical、robots、sitemapを確認します。

{% set tool_intro_what = 'HTMLやURLをもとに、SNS共有や検索結果で見える情報を確認できます。OGP画像やsitemap/robots作成も扱えます。' %} {% set tool_intro_steps = ['URLまたはHTMLを入力する', '確認したい項目を選ぶ', 'チェックする', '結果と修正ヒントを見る'] %} {% set tool_intro_notice = '外部URL確認には取得制限やタイムアウトがあります。取得できない場合はHTMLを貼り付けてください。' %} {% include 'includes/tool_intro.html' %}

使う順番

  1. URLまたはHTMLを入れる公開前ならHTML貼り付け、公開後ならURL確認が使えます。
  2. 確認項目を選ぶtitle、meta description、OGP、canonical、robots、sitemapを確認します。
  3. チェックするメタタグ、PageSpeed改善項目、sitemap/robotsを必要に応じて実行します。
  4. 修正のヒントを見る結果をコピーまたはダウンロードして、修正メモとして使えます。

1・2・3. 入力して確認する

空欄の場合は "ogp" が使用されます

このチェックリストは、PageSpeed Insights や Lighthouse の計測結果を改善するための「やること」をまとめたものです。

使い方:

  1. 対象URLで PageSpeed Insights または Lighthouse を実行して課題を確認
  2. このツールで目的や優先指標に合わせてチェックリストを生成
  3. リストに沿って対応（画像、フォント、JS/CSS、キャッシュ等）
  4. 再度計測して改善を確認

対象URLはメモ用途の任意入力です（このツール自体は計測は行いません）。

読み込んだ内容がHTMLコード欄に反映されます

⚠️ CORS制約により取得できない場合があります。その場合はHTMLを貼り付けてください。

入力したURLへサーバからアクセスしてリンクを収集します。機密サイトや社内サイトでは使用しないでください。

ベースURLを起点に同一ドメイン内をクロールしてURL一覧を取得します（最大URL数・深さで制限あり）。

パス（/で始まる）または完全なURLを入力。空行と#で始まる行は無視されます。「URLを自動収集」で取得した内容を流し込めます。

空欄の場合は設定しません

robots.txt生成

禁止したいパスを入力。空欄の場合は全許可です。

空欄の場合は baseUrl + /sitemap.xml が使用されます

4・5. 結果と修正ヒントを見る

使い方: 計測 → チェックリスト生成 → 対応 → 再計測
